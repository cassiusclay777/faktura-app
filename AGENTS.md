# AGENTS.md

Instructions for AI coding agents working on the **faktura-app** repository.

## Project Overview

Invoice helper: Next.js 15 web app with an embedded local TypeScript library (`invoice-assistant`). Users upload trip text / PDF / photo ‚Üí AI transcribes and parses ‚Üí invoice preview for print/PDF. Czech-first UI.

## Tech Stack

| Layer | Tech |
|---|---|
| Framework | Next.js 15 (App Router, `src/app/`) |
| Language | TypeScript (strict), React 19 |
| Styling | Tailwind CSS v4 (`@tailwindcss/postcss`) |
| PDF | `pdf-parse` (server-side, external package) |
| AI/Vision | Gemini (`@google/generative-ai`), Ollama (HTTP), DeepSeek (HTTP) |
| Tests | Vitest (inside `invoice-assistant/` only) |
| Lint | ESLint 9 flat config, `next/core-web-vitals` + `next/typescript` |
| Package manager | npm (lockfile committed) |
| Node | `>=20` (CI uses 22) |

No database, no ORM, no auth, no Redis/BullMQ.

## Setup & Commands

### Install

```bash
npm install
```

This also runs `invoice-assistant`'s `prepare` script ‚Üí `tsc` ‚Üí builds `dist/`.

### Environment

Copy `.env.example` to `.env` (or `.env.local`). Required keys:

- `GEMINI_API_KEY` ‚Äî for Gemini OCR/transcription and name correction
- `DEEPSEEK_API_KEY` ‚Äî for DeepSeek name correction (optional)
- Ollama vars are optional (local model)

Env load order (later overrides earlier): `invoice-assistant/.env` ‚Üí `.env` ‚Üí `.env.local`

### Dev server

```bash
npm run dev
# ‚Üí http://localhost:3000
```

### Build & start

```bash
npm run build
npm run start
```

### Lint

```bash
npm run lint
```

ESLint 9 flat config. Must pass before merge (checked in CI).

### Tests

```bash
npm run test
```

Runs `vitest run` inside `invoice-assistant/`. Tests live in `invoice-assistant/src/**/*.test.ts`.

## CI Pipeline

`.github/workflows/ci.yml` runs on every push and PR:

1. `npm ci`
2. `npm run lint`
3. `npm run build`
4. `npm run test`

All four must pass. Match this locally before pushing.

## Code Conventions

- **Path alias:** `@/*` maps to `src/*` (root tsconfig).
- **ESM imports in `invoice-assistant`:** use `.js` extensions (NodeNext resolution).
- **Client vs server:** `FakturaApp.tsx` is `"use client"`. AI/PDF logic stays in API routes and `invoice-assistant`.
- **Error handling:** API route returns JSON `{ error }` on failure. Client uses `readJsonResponse` + `formatUnknownError`.
- **UI language:** Czech. Strings, labels, and error messages are in Czech.
- **Print styles:** `print:` Tailwind utilities and `@media print` rules in `globals.css` for invoice output.
- **No Prettier config** ‚Äî follow existing formatting conventions in surrounding code.
- **Strict TypeScript** ‚Äî no `any` unless unavoidable.
- **`pdf-parse`** is listed in `serverExternalPackages` in `next.config.mjs` ‚Äî don't remove this or dev server/build will break.
- **`transpilePackages: ["invoice-assistant"]`** in `next.config.mjs` ‚Äî required for the local package to work with Next.

## Cursor Cloud Specific Instructions

### Running the dev server

```bash
npm install && npm run dev
```

The app serves at `http://localhost:3000`. No database or external services needed for basic UI testing. AI features require API keys in `.env`.

### Validating changes

Before committing, always run:

```bash
npm run lint && npm run build && npm run test
```

This mirrors the CI pipeline.

### Manual testing

- For UI changes: start the dev server and test at `http://localhost:3000`.
- The app is a single-page app with one main component (`FakturaApp.tsx`). All UI interaction happens on the root page.
- Invoice preview and print styling can be tested via the browser print dialog.
- API route testing: `POST /api/process` with `multipart/form-data` (file + provider field). Requires valid API keys for AI features.

### Adding dependencies

- Root deps: `npm install <package>` from `/workspace`
- `invoice-assistant` deps: `npm install <package>` from `/workspace/invoice-assistant`
- After changing `invoice-assistant` source, rebuild: `cd invoice-assistant && npm run build`

### Working with `invoice-assistant`

This is a local package linked via `"file:./invoice-assistant"`. After editing its source:

1. Rebuild: `cd invoice-assistant && npm run build`
2. The Next.js app picks up changes via `transpilePackages` ‚Äî a dev server restart may be needed.
3. Run its tests: `cd invoice-assistant && npx vitest run`
